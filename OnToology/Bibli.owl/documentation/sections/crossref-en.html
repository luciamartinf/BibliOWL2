Access conditionc back to ToC or Class ToC

IRI: http://dati.beniculturali.it/cis/AccessCondition

Concept used to represent general information for the public access to the Cultural Institute or Site or to the Event. For instance, examples of information include: opening hours, type of admission, temporary closing, etc.
This class is used for representing the access conditions associated with a 'Cultural Institute or Site' or an 'Event'. Access conditions are meant to be indivuduals of the following classes: * Opening Hours Specification * Temporary Closure * Type of Booking An example in turtle is the following: <pre> cis a CulturalInstituteOrSite ; description "An entity representing an individual of the class 'Institute or Site.'" ; hasAccessCondition cis_access_condition . cis_access_condition a Access ; description "The individual of the class 'Access' that specifies the access condition associated with the CulturalInstituteOrSite called cis" . </pre>
